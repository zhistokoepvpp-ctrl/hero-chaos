## 1. КОНЦЕПЦИЯ

**Hero Chaos** — онлайн-игра на 8 игроков (P2P, один хостит). Каждый выбирает героя, проходит волны монстров в личной арене, прокачивается и покупает предметы в общем лобби. Каждые 5 волн — 1v1 дуэли. Каждые 10 — босс. Последний выживший — чемпион.

### Ключевые цифры
| Параметр | Значение |
|----------|----------|
| Игроков | 8 |
| Героев (MVP) | 10 |
| Макс. уровень | 30 |
| Таймер лобби | 60 сек (реди-система) |
| Таймер волны | 60 сек (+ овертайм) |
| Таймер дуэли | 60 сек на пару |
| Жизней | 2 (3-я смерть = вылет) |
| Слотов предметов | 6 |
| Базовых предметов | 15 |
| Комбинированных | 10 |
| Расходников | 5 |
| Типов монстров | 7 |
| Боссов | 3 |

## 2. ПОЛНЫЙ GAME LOOP

```
1. Хост создаёт комнату (UPnP + рега на мастер-сервере)
2. 8 игроков подключаются (мастер-сервер / IP вручную)
3. Герой-селект (30 сек, free pick, no dupes)
4. ЛОББИ (реди-система, макс 60 сек):
   - Шоп (купить/продать/combine)
   - Атрибуты (+STR/+AGI/+INT)
   - Ставки (если следующая волна = дуэль)
   - Чат (Enter, 10 сек видимость)
5. ВОЛНА (60 сек):
   5a. Монстры: каждая не-5-я и не-10-я волна
   5b. Дуэли: каждая 5-я волна
   5c. Босс: каждая 10-я волна
6. РЕЗУЛЬТАТЫ (5 сек) → статистика волны
7. GOTO 4, пока не останется 1 игрок
8. GAME OVER → таблица + MMR
```

### NAT Punchthrough

```
1. Хост регает комнату (сервер видит его IP)
2. Игрок запрашивает комнату
3. Сервер говорит обоим: "пошлите UDP друг другу"
4. Оба шлют → NAT открывает проход → P2P работает
```

## 14. АНТИЧИТ (максимальный уровень)

### Сервер проверяет
- **Gold/XP:** любое изменение — только через сервер
- **Покупки:** есть ли золото? Есть ли слот?
- **Убийства:** не больше чем монстров на волне
- **Позиция:** скорость не выше макс
- **Кулдауны:** способность нельзя использовать раньше CD
- **Уровень:** не выше 30

### Rate Limiting
- ≤20 команд/сек с клиента

### Session Token
- Уникальный токен на сессию
- Все RPC с токеном

### Нарушение → Kick
- 1-е: warning + log
- 2-е: kick (AI-бот)
- Многократно: бан на мастер-сервере

## 15. AI БОТ (при дисконнекте)

### Включение
- Таймаут 15 сек → AI берёт управление

### Поведение (в порядке приоритета)
1. Если HP<30% → Health Potion + отступить
2. Искать ближайшего врага → двигаться к нему
3. Использовать Q/W по ротации (зависит от героя)
4. Если врагов нет → стоять

### В лобби
- Не покупает, не качает, не ставит
- Auto-ready через 30 сек

### Reconnect
- Игрок может переподключиться в течение 3 минут
- Полный state sync при reconnect
- После 3 минут → reconnect невозможен

## 19. АРХИТЕКТУРА GODOT

### Scene Tree (во время игры)

```
Game (autoload GameManager)
├── Lobby.tscn
├── Arena.tscn (instanced per player)
├── DuelArena.tscn
├── SpectatorUI.tscn
└── ResultsUI.tscn
```

### Autoload синглтоны

| Синглтон | Задача |
|----------|--------|
| GameManager | State machine (Lobby/Wave/Duel/Results) |
| NetworkManager | ENet server/client, RPCs, sync |
| DataManager | profile.json save/load |
| MasterServerAPI | HTTP запросы к мастер-серверу |
| ItemDatabase | Все предметы (Resources) |
| HeroDatabase | Все герои (Resources) |
| MonsterDatabase | Все монстры |
| AudioManager | BGM + SFX |
| AntiCheat | Валидация действий |
| Constants | Все константы |

### Сеть (Godot 4 ENet)
- `ENetMultiplayerPeer` — транспорт
- `MultiplayerSynchronizer` — sync позиций
- `rpc()` — команды
- `MultiplayerSpawner` — спавн в общих сценах

### Авторитетность
- Сервер: экономика, таймеры, дуэли, жизни, фазы
- Клиент: своя арена (монстры, бой)
- AntiCheat: валидация рапортов от клиента

## 20. ПРОИЗВОДИТЕЛЬНОСТЬ

- Каждый клиент симулирует только свою арену
- Хост: сервер-логика + своя арена. Минимальная доп. нагрузка
